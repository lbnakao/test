# 松岡製作所 Instagram カルーセル投稿 自動運用ツール

毎週水曜投稿の画像5〜6枚カルーセル＋ポエティックなキャプションを半自動で準備する Claude Code 用プロジェクト。

---

## ユーザーが Claude Code に話しかけたら即応

仲尾さんから「今週分のインスタ準備して」と言われたら、本ドキュメントに従って下記10ステップを実行する。

---

## 🚨 重要：PowerShellスクリプト呼び出しの絶対ルール【守らないと運用破綻】

このプロジェクトのスクリプト実行は **下記の形式の1択のみ** を使う。他の形式を試すな。これらは settings.json で事前許可済みなので、ユーザーに権限ダイアログが出ない。

**✅ 必ずこの形で呼ぶ:**
```
powershell -ExecutionPolicy Bypass -File "<絶対パス>\scripts\<スクリプト名>.ps1" <引数>
```

**❌ 試すな（毎回失敗する／許可ダイアログを出す）:**
- `pwsh.exe -File ...` → pwshはインストールされていない
- `& "...ps1"` → 実行ポリシーで弾かれる
- `pwsh -File ...` → 同上

**ファイル書き込みも以下の形に固定する:**
- デスクトップ出力フォルダへの書き込みは `Write` ツールではなく **`powershell -Command "Set-Content -Path '...' -Value '...' -Encoding utf8"`** を使う（Writeツールはユーザー領域の書き込みで権限ダイアログを出す）

## 実行手順（必ずこの順番）

### 1. 状態を読む
- `state.json` を読む
- `last_kitchen` から `config.json.rotation_order` の次のキッチンID を求める
- ローテーション順（`01 → 03 → 02 → 04 → 01 ...`／03と04は同一人物なので連続させない）

### 2. 未使用画像のリストを取得
PowerShellスクリプトを呼ぶ（**必ずこの形・上記の絶対ルール参照**）：
```
powershell -ExecutionPolicy Bypass -File "C:\Users\event\OneDrive\デスクトップ\リベルブライトのAI\test\instagram-tool\scripts\list_unused.ps1" -KitchenId <next_id>
```
- 標準出力に未使用のファイル名（拡張子なし）を改行区切りで出す

### 3. サンプル画像を閲覧してシーン分類【**並列・8枚厳守**】
- 未使用一覧から **ちょうど8枚**（多すぎず少なすぎず）をファイル番号がバラけるよう均等サンプリング
- **必ず1メッセージ内で8枚すべてを Read tool で並列読み込み**（順次読みは禁止＝大幅な時短のため）
- 8枚を以下シーンに振り分ける：
  - **引き**：キッチン全景・モデル＋空間を両方収める引きの構図
  - **食事**：食事中（朝食シーン）・コーヒーなど
  - **質感**：シンクや天板のステンレス質感が主役（手元アップ含む）
  - **調理準備**：コンロ前・食材を扱う・鍋を持つ
  - **収納**：扉を引き出す、開け閉め・パントリーを使う
  - **清掃**：拭く・磨く・洗う
- 8枚で5シーンが揃わない場合のみ、追加で2〜4枚を**並列で**追加Read（順次禁止）

### 4. 5〜6枚を選定
- 同じシーンが2枚にならないように、最低5シーンをカバー
- 縦・横のバランス（1枚目は引き＝横画面推奨）
- 視覚的に物語が繋がる順番に並べる：
  - **1枚目**：引きの全景（フック）
  - **2-4枚目**：暮らしの所作（食事／質感／調理準備）
  - **5-6枚目**：機能・整いの所作（収納／清掃）

### 5. 選定結果を表示（確認待ちなしで次へ）
選定した6枚（順番付き）と各シーン名を表に整理して**画面に表示**する。
**確認は待たず、そのまま次のステップへ進む**（仲尾さんが結果を見て不満なら「やり直して」と指示する運用）。

### 6. キャプション生成
- `reference/post_DXg4mJ8gX5G.txt` と `reference/post_DVedtvKgT8R.txt` を読んで文体を確認
- 構成（必須）：
  ```
  「フード」

  日本語本文（ポエティック、5〜8行）

  英訳タイトル
  英訳本文

  #松岡製作所
  #タグ2
  #タグ3
  #タグ4
  #タグ5
  ```
- ステンレスの清潔感・オーダーの自由度・ヒヤリハットインスペレーション付与等に**軽く**触れる
- 商品スペックを語らない、物語フックを少し詩的に
- フードは「朝の所作」「夜の場所」「暮らしの整い」など、暮らしと所作にフォーカス

### 7. デスクトップに出力
今日の日付で出力先パスを決定：
```
$dst = "C:\Users\event\OneDrive\デスクトップ\松岡カルーセル_YYYY-MM-DD"
```

PowerShellで画像コピー（**必ずこの形**）：
```
powershell -ExecutionPolicy Bypass -File "C:\Users\event\OneDrive\デスクトップ\リベルブライトのAI\test\instagram-tool\scripts\copy_photos.ps1" -KitchenId <id> -FileNames @("TJA01XXX.jpg","TJA01YYY.jpg",...) -DestinationFolder "$dst"
```

### 8. caption.txt を出力先に保存
`{dst}\caption.txt` にキャプション本文＋投稿メタ情報を書き込む。

**Writeツールは絶対パス（`C:\Users\event\...`）で呼ぶこと。`~\...` や相対パスはWindowsで展開されず失敗する。** settings.jsonの `Write(C:\\Users\\event\\OneDrive\\デスクトップ\\松岡カルーセル_*\\**)` ルールでマッチするので権限ダイアログは出ない。

### 9. state.json を更新
```
{
  "last_post_date": "YYYY-MM-DD",
  "last_kitchen": "<id>",
  "used_files": { "01": [...既存..., "新しく使ったファイル名(拡張子なし)"] }
}
```

### 10. アーカイブ
`archive/YYYY-MM-DD_<kitchen_name>/` フォルダを作成し、caption.txt と選定リスト（manifest.txt）をコピー保存。

### 11. Slack DM 送信【必須・最終ステップ】
Slack MCPツール `mcp__claude_ai_Slack__slack_send_message` を使い、`channel_id="U0997A7C8MV"`（仲尾光平本人へのDM）に下記フォーマットでメッセージを送信する：

```
*🍳 松岡製作所 今週のカルーセル準備完了*

📅 投稿予定日: YYYY-MM-DD（曜日）
🏠 キッチン: <キッチン名>
📁 出力先: `C:\Users\event\OneDrive\デスクトップ\松岡カルーセル_YYYY-MM-DD`

*選定画像（順番）*
1. <ファイル名> — <シーン>
2. ...
6. ...

*キャプション*
```
<生成したキャプション本文を全文>
```

✅ Instagramへの投稿は手動で実施してください。
```

エラー時（G:ドライブ無し・画像枯渇等）は、エラー内容を同じDMに送信して終了する：
```
*⚠️ 松岡製作所 自動準備エラー*
内容: <エラー詳細>
対処: <推奨アクション>
```

## 文体の鉄則
- 「制作会社っぽい」機能訴求はしない
- 「暮らしの中の所作」「素材の手触り」「光と陰影」を主役に
- 1人称や本人名は最小限。風景と動作を淡々と描く
- 英訳は意訳でOK。日本語の余韻が出る訳に
- ハッシュタグは5個。`#松岡製作所` 必須、残り4個はその回のフードに合わせる

## 🚨 キャプション生成の絶対原則【守らないと事実誤認の文が出る】

### 1. 画像で実際に見えるものだけを書く
**画像に映っていない物・行為・状態を絶対に書かない**。これまで以下のような捏造があり、仲尾さんから却下された：
- ❌ 「湯気がのぼる前の」← 湯気のシーンが画像にないのに書いた
- ❌ 「冷えたスコーン」← スコーンが映ってないのに書いた
- ❌ 「ステンレスの天板」← 04アメリカンキッチン（人工大理石）に書いた

書く前に、選定6枚で **実際に何が映っているか** を再確認し、それだけを描写する。映っていない食材・行為・天候・湯気・香り等は書かない。

### 2. キッチンの素材を間違えない
config.json の `features` と `avoid_words` を **必ず参照** すること：
- **01 ハイエンドキッチン**：ステンレスモザイクタイル、ウォルナット
- **02 手元隠せるアイランドキッチン**：波模様木目、ブルーグレー、腰壁
- **03 オールステンレスキッチン**：全面ステンレス、ヘアライン
- **04 アメリカンキッチン**：**白サブウェイタイル・人工大理石天板**（ステンレスではない！）

`avoid_words` に列挙された語は **そのキッチンでは絶対に使わない**。

### 3. キャプション執筆フロー（必須）
1. 選定6枚の Read で見えたものを **メモ書きで列挙**（食べ物・動作・素材・小物・光）
2. config.json の features と avoid_words を確認
3. メモに書いたものだけを材料に、文体ルールに沿って詩的に書く
4. 完成後、自分でセルフチェック：「この一行は画像に映っているか？」「この素材はこのキッチンに存在するか？」

## エラー時の対処
- `G:` ドライブが見つからない → KIOXIA SSD接続を促す
- 未使用画像が10枚未満 → 仲尾さんに通知。ローテーションを飛ばすか、未使用を維持するか相談
- 全キッチン枯渇したら、used_filesをリセットして再開可（仲尾さんに確認）
